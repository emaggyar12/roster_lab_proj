Optimizer
Runtime source:
- frontend_site/lib/optimizer.ts
- frontend_site/components/RosterOptimizer.tsx

Underlying data source:
- Same combined `players` array from frontend_site/data/players.ts.
- No separate external optimizer data file is loaded at runtime.

Candidate sources:
- Transfers from getPortalPlayers():
  - player_source = transfer
  - is_in_portal = true
  - draft_status is false/undefined
- High-school recruits from getHsPlayers():
  - player_source = hs
  - draft_status is false/undefined
- Returning/current roster players from selected team roster:
  - player_source = roster

Optimizer normalization fields:
- optimizer_player_id:
  - transfer:{transfer_barttorvik_trid}
  - hs:{hs_player_key}
  - returning:{returning_bvt_pid or returning_barttorvik_trid}
- position_group:
  - normalized from position or top playtype
  - G/F/C
- projected_bpr:
  - transfer: transfer_bpr, fallback projected_bpr
  - HS: hs_bpr, fallback projected_bpr
  - returning: projected_bpr
- skill fields:
  - skill_spacing_percentile
  - skill_facilitating_percentile
  - skill_rim_protection_percentile
  - skill_defense_percentile
  - skill_finishing_percentile

Optimizer-derived data points:
- team skill ratings:
  - BPR-weighted average of skill percentiles
  - denominator is sum(abs(projected_bpr))
  - fallback is simple average if denominator <= 0
- total rating:
  - sum of the five team skill ratings
- weakest rating:
  - minimum of the five team skill ratings
- rating changes:
  - final team rating minus current team rating for each skill
- individual total gain:
  - total rating after adding one candidate minus current total rating
- individual weakest gain:
  - weakest rating after adding one candidate minus current weakest rating
- added BPR:
  - sum of selected players projected_bpr
- recommendation ordering:
  - sort modes in optimizer.ts:
    - Best Overall Fit
    - Total Gain
    - Weakest Category Gain
    - Added BPR
    - category-specific gain modes
